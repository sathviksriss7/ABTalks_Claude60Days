Credit Card Fraud Detection
Week 4 · Day 26 · Classification + Class Imbalance
Tackle one of the most common real-world DS challenges — extreme class imbalance. Build a fraud classifier with XGBoost and handle the 99:1 class ratio correctly, a critical skill for healthcare payment integrity.
XGBoostSMOTE AUC-ROCPrecision-Recall Imbalanced-learn
Steps
  1. Load Kaggle Credit Card Fraud dataset
  2. Show why accuracy is misleading here
  3. Apply SMOTE oversampling on training set only
  4. Train XGBoost with class_weight parameter
  5. Evaluate with Precision-Recall curve + F1 score

Healthcare Claims Analytics (Domain Capstone)
Week 4 · Day 28 · Full Stack DS + Your Domain
Your differentiator project. Use your healthcare AI background to build something no generic DS bootcamp grad has — a claims analytics model using SQL + Pandas + ML. Denial prediction or cost segmentation both work great here.
SQLPandas XGBoostDomain Knowledge GenAI-assisted
Steps
  1. Use CMS public data or generate synthetic claims data
  2. SQL-level aggregation (denial rates, cost by procedure)
  3. Feature engineer from claims fields (ICD codes, payer type)
  4. Train classifier: predict denial vs approval
  5. Write executive summary tying to real payer business value
